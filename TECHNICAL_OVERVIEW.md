# 🔧 Error Code RAG 系統 - 技術架構與用量分析

## 📊 系統架構總覽

```
┌─────────────────────────────────────────────────────────────┐
│                        使用者瀏覽器                          │
├─────────────────────────────────────────────────────────────┤
│  1️⃣ 使用者輸入自然語言查詢                                   │
│     「相機測試失敗」                                         │
│                          ↓                                  │
│  2️⃣ Transformers.js (本地 AI)                             │
│     • 模型：Xenova/paraphrase-multilingual-MiniLM-L12-v2   │
│     • 輸入 → Embedding 向量 (384維)                         │
│     • 純本地運算，零成本                                   │
│                          ↓                                  │
│  3️⃣ 語意搜尋 (Cosine Similarity)                          │
│     • 與 1,217 筆預計算向量比對                            │
│     • 找出相似度最高的 Top-10                              │
│     • 本地計算，零成本                                     │
│                          ↓                                  │
│  4️⃣ GAS Proxy 轉發                                        │
│     • 送給 Groq API (Llama 3)                              │
│     • 產生 AI 分析與建議                                   │
│     • 每次約消耗 1,500 tokens                              │
│                          ↓                                  │
│  5️⃣ 顯示結果                                               │
│     • AI 分析 + 相關 Error Code 列表                       │
└─────────────────────────────────────────────────────────────┘
```

## 🧮 用量與成本分析

### 📦 Transformers.js (瀏覽器端)

| 項目 | 數值 |
|------|------|
| **模型大小** | ~50MB (首次下載) |
| **執行位置** | 使用者瀏覽器本地 |
| **費用** | **$0 (完全免費)** |
| **運算資源** | 使用者電腦 CPU/GPU |
| **首次載入時間** | 10-20 秒 (依網速) |
| **後續載入** | 瀏覽器快取，瞬間 |

### 🔍 語意搜尋 (本地計算)

| 項目 | 數值 |
|------|------|
| **資料量** | 1,217 筆 Error Code |
| **向量維度** | 384 維 |
| **計算方式** | Cosine Similarity |
| **費用** | **$0 (完全免費)** |
| **搜尋時間** | < 100ms (即時) |

### 🤖 Groq API (LLM 分析)

| 項目 | 數值 |
|------|------|
| **模型** | llama-3.1-8b-instant |
| **每次查詢消耗** | ~1,500 tokens |
| **免費版額度** | **1,000,000 tokens / 月** |
| **每月可查詢次數** | **約 666 次** |
| **每天平均** | **約 22 次** |

### ☁️ GAS Proxy (Google Apps Script)

| 項目 | 數值 |
|------|------|
| **執行次數限制** | 每日 20,000 次 (免費版) |
| **資料傳輸** | ~100KB / 次 |
| **費用** | **$0 (完全免費)** |

## 📈 系統效能指標

### 載入效能

| 階段 | 時間 | 說明 |
|------|------|------|
| **首次開啟** | 10-20 秒 | 下載 AI 模型 (50MB) |
| **二次開啟** | 1-2 秒 | 使用瀏覽器快取 |
| **資料同步** | 1-2 秒 | 從 GAS 抓取最新資料 |
| **搜尋響應** | < 100ms | 本地語意比對 |
| **AI 分析** | 1-3 秒 | Groq API 回應 |

### 資料流向

```
使用者開啟網頁
    ↓ 1-2 秒
從 GAS 抓取 1,217 筆資料 (100KB)
    ↓
與本地 Embeddings 合併 (9.8MB)
    ↓
⏱️ 系統就緒，可開始搜尋

每次搜尋：
使用者輸入 → 本地產生向量 (~500ms) → 本地比對 (<100ms) → 送 Groq (1-3s)
```

## 🔒 資料安全與隱私

### 資料儲存位置

| 資料類型 | 儲存位置 | 說明 |
|---------|---------|------|
| **Embeddings** | GitHub Pages | 公開，但僅是向量數字 |
| **Error Code 資料** | Google Sheet + GAS | 受 Google 保護 |
| **使用者查詢** | 無儲存 | 即時處理，不留記錄 |
| **API Key** | GAS Properties | 加密儲存，前端看不到 |
| **快取資料** | 使用者 localStorage | 僅存在使用者瀏覽器 |

### 安全設計

```
瀏覽器 ──→ GAS Proxy ──→ Groq API
  ↑           ↑            ↑
無法直接    API Key 在    看不到
存取 Key    這裡安全      使用者資料
```
